## âœ… **Key Principles**

### **QA Mindset:**
1. **Test beyond requirements** - Discover, don't just verify
2. **Document everything** - Evidence is crucial
3. **Question assumptions** - If it seems wrong, it probably is
4. **Think like users** - Consider real-world scenarios
5. **Protect quality** - You're the last line of defense

### **Reporting Principles:**
1. **Be specific** - Clear steps to reproduce
2. **Provide evidence** - Screenshots, logs, videos
3. **Assess impact** - How does this affect users?
4. **Be professional** - Clear, concise, collaborative
5. **Follow up** - Verify fixes actually work

## ðŸ“‹ **Quick Reference**

### **Issue Priority Levels:**
- **CRITICAL**: Blocks core functionality, data loss, security breach
- **HIGH**: Major feature broken, difficult workaround
- **MEDIUM**: Feature partially broken, workaround exists
- **LOW**: Cosmetic, minor inconvenience, easy workaround

### **Issue Types:**
- **Bug**: Something doesn't work as intended
- **Enhancement**: Improvement to existing functionality
- **Standards Violation**: Doesn't meet accessibility/security/UX standards
- **Question**: Unclear if behavior is correct

### **Testing Checklist (Quick):**
- [ ] Functionality (happy path + edge cases)
- [ ] Security (injection, validation, auth)
- [ ] Accessibility (keyboard, contrast, alt text)
- [ ] Usability (clear, intuitive, consistent)
- [ ] Performance (fast, responsive)
- [ ] Responsive (desktop, tablet, mobile)
